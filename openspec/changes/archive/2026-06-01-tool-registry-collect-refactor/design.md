# Design: Tool Registry + Collect Refactoring

## Architecture

```
┌─────────────────────────────────────────────────────────┐
│                    Tool Discovery Layer                  │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  tool-registry skill                                    │
│  ┌─────────────────────────────────────────┐            │
│  │ 1. Probe built-in tools (WebSearch,     │            │
│  │    WebFetch — always available)          │            │
│  │ 2. Check MCP servers (ListMcpResources) │            │
│  │ 3. Check CLI tools (opencli, etc.)      │            │
│  │ 4. Write ~/.claude/rules/               │            │
│  │    tool-registry.yaml                   │            │
│  └─────────────────────────────────────────┘            │
│                      ↓                                  │
│  tool-registry.yaml (discovered capabilities)           │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                    Collect Pipeline                      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  collect skill                                          │
│  ┌─────────────────────────────────────────┐            │
│  │ 1. Read tool-registry.yaml              │            │
│  │ 2. Select search tools by priority      │            │
│  │ 3. Select fetch tools by priority       │            │
│  │ 4. Execute search + fetch + curate      │            │
│  │    (merged collect+curate in one pass)  │            │
│  │ 5. Output curated files to 0-inbox/     │            │
│  └─────────────────────────────────────────┘            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## Tool Registry Format

```yaml
# ~/.claude/rules/tool-registry.yaml
# Auto-generated by tool-registry skill.

metadata:
  generated_at: "2026-06-01T10:00:00Z"
  generator: tool-registry skill v1.0

search_tools:
  - name: WebSearch
    type: builtin
    priority: 1
    status: available
    description: "Claude built-in web search"
    capabilities: [general_search, news, realtime]

  - name: opencli
    type: cli
    priority: 2
    status: available  # or: not_installed, error
    description: "opencli search command"
    capabilities: [general_search, deep_content]
    command: "opencli search '{query}' --format json"

  - name: acgithub
    type: mcp
    server: acgithub
    priority: 3
    status: available
    description: "GitHub code and repository search"
    capabilities: [code_search, repos, documentation]
    methods: [search_code, search_repositories]

fetch_tools:
  - name: WebFetch
    type: builtin
    priority: 1
    status: available
    description: "Claude built-in page fetcher"
    capabilities: [url_fetch, text_extraction]

  - name: opencli
    type: cli
    priority: 2
    status: available
    description: "opencli browser automation"
    capabilities: [browser_fetch, js_rendering, spa_support]
    command: "opencli web get '{url}' --format json"
```

## Collect Skill Refactored Logic

```
collect skill receives: { topic, sources, depth }

1. Read ~/.claude/rules/tool-registry.yaml
   - If missing or stale (>7 days) → trigger tool-registry skill first

2. For each source in sources[]:
   a. Select search tool (filter by status=available, sort by priority)
   b. Execute search with selected tool
   c. Collect URLs from results

3. For each unique URL:
   a. Select fetch tool (filter by status=available, sort by priority)
   b. Fetch content
   c. Save to 0-inbox/{topic}/source-{n}.md

4. Curate inline (merged from curate skill):
   - Score each file (1-5) on authority, freshness, completeness, readability
   - Mark duplicates for removal
   - Classify into core_concepts, practical_examples, advanced

5. Remove low-quality and duplicate files
6. Save curation metadata to 0-inbox/{topic}/metadata.yaml
```

## Fallback Strategy

```
Priority chain for search:
  WebSearch (always available) → opencli (if installed) → acgithub (MCP, if connected)

Priority chain for fetch:
  WebFetch (always available) → opencli (if installed, for SPA) → skip (if URL unreachable)

If ALL tools fail:
  → Log error to error-log.md
  → Ask user to provide URLs manually
```

## Performance Impact

| Metric | Before | After | Savings |
|--------|--------|-------|---------|
| Subagent calls | 4 (collect, curate, write, beautify) | 3 (collect+curate, write, beautify) | 1 subagent |
| Token overhead | ~15-20k per session | ~10-14k per session | ~5-8k tokens |
| Time | ~2-4 min | ~1.5-3 min | ~30-60s |
| Tool flexibility | opencli only | WebSearch + opencli + MCP | 3x tools |

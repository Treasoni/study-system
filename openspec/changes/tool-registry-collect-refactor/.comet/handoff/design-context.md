### Problem
- collect skill 硬绑定 opencli → 单点故障
- WebSearch/WebFetch/MCP 能力被浪费
- 用户必须先装 opencli 才能用学习系统

### Goal
- 创建工具发现层：auto-discovers search/fetch tools
- 写入 `~/.claude/rules/tool-registry.yaml`
- collect 技能动态选择最佳工具

### Scope
- In: tool-registry 新技能 + collect 重构 + collect/curate 合并
- Out: opencli 本身、新 MCP 服务器、write/beautify 管道

### Architecture
```
tool-registry skill → 扫描工具 → 写入 YAML
collect skill → 读取 YAML → 按优先级选择工具 → 搜索+抓取+curate
```

### Registry Format
- search_tools: WebSearch(builtin,p1) → opencli(cli,p2) → acgithub(mcp,p3)
- fetch_tools: WebFetch(builtin,p1) → opencli(cli,p2)
- 每个工具: name, type, priority, status, capabilities, command

### Collect Refactored Logic
1. 读取 tool-registry.yaml
2. 按优先级选择搜索/抓取工具
3. 执行搜索+抓取（合并 curate 逻辑：评分、去重、分类）
4. 输出到 0-inbox + metadata.yaml

### Fallback
- 搜索: WebSearch(永远可用) → opencli → acgithub
- 抓取: WebFetch(永远可用) → opencli(SPA) → skip
- 全部失败: 记录错误，请求用户手动提供 URL

# Skill: evaluate（评估效果）

## 触发时机
用户审核通过 beautify 产出后（可选阶段）。

## 输入
`{OUTPUT_PATH}/{topic}/{topic}.md`

## 五个评估维度（各 0-10 分）

### 1. 完整性
核心概念是否覆盖全面？
- 10：全面覆盖，有深度
- 7-9：主要话题覆盖，少量缺口
- 4-6：部分重要话题缺失
- 0-3：严重覆盖不足

### 2. 准确性
内容是否与原始资料一致？
- 抽查 3-5 个关键观点，与整理资料交叉验证
- 10：全部通过 / 7-9：轻微不精确 / 4-6：部分不准确 / 0-3：严重错误

### 3. 可读性
是否易于理解？
- 10：结构清晰 / 7-9：基本清晰 / 4-6：部分难跟上 / 0-3：结构混乱

### 4. 实用性
有无可操作的示例或指南？
- 10：丰富示例 / 7-9：较好的示例 / 4-6：不够实用 / 0-3：纯理论无示例

### 5. 关联性
双链和交叉引用是否准确？
- 10：丰富的知识链接 / 7-9：较好 / 4-6：有但不够 / 0-3：几乎没有链接

## 执行步骤

1. **阅读最终笔记**：读取 `{OUTPUT_PATH}/{topic}/{topic}.md`
2. **交叉验证**：读取 `{SYSTEM_ROOT}/1-curated/{topic}/core-concepts.md`，抽查 3-5 个观点
3. **逐维度评分**：5 个维度分别打分并记录观察
4. **计算总分**：`总分 = 完整性 + 准确性 + 可读性 + 实用性 + 关联性`
5. **判定等级**：
   - ≥ 40 且所有维度 ≥ 5：优秀 — 可以直接使用
   - 30-39：良好 — 建议小幅修改
   - < 30 或任一维度 < 5：需要返工
6. **生成改进建议**：对 < 7 分的维度给具体建议

## 产出
写入 `{SYSTEM_ROOT}/4-meta/evaluation/`：
- `{topic}-eval.md`：含各维度评分、详细发现、改进建议、验证过的观点列表

## 禁止行为
- 不要在评估过程中修改笔记
- 不要虚高或虚低评分
- 不要跳过与原始资料的交叉验证
- 不要给笼统反馈而没有具体例子
